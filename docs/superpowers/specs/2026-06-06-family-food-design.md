# 萌萌家厨 — 家庭点菜小程序 设计文档

> 日期：2026-06-06
> 状态：待用户审核
> 作者：Claude

## 1. 概述

### 1.1 背景
家庭场景中，谁做饭、做什么菜经常需要沟通。设计一个微信小程序，让家人可以浏览菜品、下单点菜、查看彼此的点单情况。

### 1.2 目标
- 让家庭成员在手机上点自己想吃的菜
- 在同一台设备上支持多账号切换（模拟多人）
- 全部菜品附带图片，UI 风格统一粉嫩可爱

### 1.3 范围
- **包含**：账号系统、菜品管理（5 分类 + 自定义）、点单、订单查看、个人中心
- **不包含**：云同步、消息推送、支付、外卖配送

## 2. 用户与场景

### 2.1 目标用户
家庭成员（父母、孩子、老人），3-6 人核心家庭。

### 2.2 核心场景
1. **小明的妈妈**登录后浏览「家常菜」分类，看到「番茄炒蛋」点击查看详情
2. **小明的爸爸**登录后添加一道新菜「可乐鸡翅」，填写图片（用 emoji 占位）
3. **小明的姐姐**登录后点了几道菜作为晚餐订单，提交
4. **小明**晚上登录，查看「今日订单」看到大家点的菜

### 2.3 设备约束
- 微信内运行的小程序
- 本地存储，不依赖网络
- 单设备多账号切换

## 3. 功能需求

### 3.1 账号系统

| 功能 | 描述 |
|------|------|
| 注册 | 用户名（唯一）+ 密码（≥6 位）+ 昵称 + 头像 emoji |
| 登录 | 输入用户名和密码登录 |
| 登出 | 清除当前用户登录态 |
| 切换账号 | 登出后可重新登录其他账号 |

**约束**：
- 密码明文存储在本地（产品定位为家庭玩具级，不做安全加密）
- 用户名 2-16 字符，密码 6-20 字符
- 昵称 1-12 字符
- 头像从预设 emoji 列表中选 1 个（🍚🍜🥗🍎🥤🍕🍔🌽🍓🍇🥑🍣等）

### 3.2 菜品管理

#### 3.2.1 分类
固定 5 个分类（不可增删）：
- 🍚 家常菜
- 🍲 汤
- 🥞 早餐
- 🍎 水果
- 🥤 饮品

#### 3.2.2 菜品字段
```javascript
{
  id: string,            // uuid
  name: string,          // 菜名，1-20 字符
  category: string,      // 5 分类之一
  emoji: string,         // 用作图片占位的 emoji
  description: string,   // 描述，0-100 字符
  tags: string[],        // 标签，如 ["辣", "清淡", "小孩爱吃"]
  createdBy: string,     // 创建者用户名
  createdAt: number,     // 时间戳
  isPreset: boolean      // 是否预置（预置的预置菜可编辑但不可删）
}
```

#### 3.2.3 操作
- **浏览**：按分类切换 Tab 查看菜品卡片
- **查看详情**：点击菜品查看完整信息
- **添加**：填写表单新增菜品，emoji 从选择器中选
- **编辑**：修改菜品信息（仅创建者或预置菜可编辑）
- **删除**：仅创建者自己的菜可删除，预置菜不可删

### 3.3 点单系统

#### 3.3.1 点单流程
1. 用户在「菜单」Tab 浏览/选择菜品，点击 ➕ 加入点单
2. 跳转到「点单」Tab，购物车式展示已选菜品
3. 调整每道菜的份数（1-10）、口味备注（0-30 字符）
4. 选择用餐时间（早餐/午餐/晚餐 + 可选具体时间）
5. 提交订单

#### 3.3.2 订单字段
```javascript
{
  id: string,
  userId: string,           // 下单人用户名
  userNickname: string,
  userEmoji: string,
  items: [
    {
      dishId: string,
      dishName: string,
      dishEmoji: string,
      quantity: number,      // 份数
      note: string           // 备注
    }
  ],
  mealType: string,         // 早餐/午餐/晚餐
  mealTime: number,         // 用餐时间戳（可选）
  status: string,           // 待定/已确认/已上桌/已取消
  createdAt: number,
  remark: string            // 整单备注，0-100 字符
}
```

#### 3.3.3 状态机
- `待定` → `已确认` / `已取消`
- `已确认` → `已上桌` / `已取消`
- `已上桌` / `已取消` 为终态

### 3.4 订单查看

| 视图 | 描述 |
|------|------|
| 今日订单 | 当前日期的所有订单，按时间倒序 |
| 历史订单 | 过去日期的所有订单，可按月分组 |
| 订单详情 | 完整信息：点单人、菜品、备注、状态、操作按钮 |

**操作**：
- 当前用户只能修改自己提交的订单
- 状态可由创建者流转：待定→已确认→已上桌

### 3.5 个人中心

- 头像 emoji + 昵称 + 用户名
- 统计：累计点单数、累计菜品数
- **我的常点菜**：从历史已上桌订单中累计每道菜的份数总和，取 top 3
- **我的订单**：跳转订单列表
- **切换账号**：登出回到登录页
- **关于**：版本号、说明

### 5.1 配色

| 用途 | 颜色 | 十六进制 |
|------|------|---------|
| 主色 | 樱花粉 | `#FFB6C1` |
| 辅色 | 浅粉 | `#FFC0CB` |

| 强调/成功 | 苹果绿 | `#7FCD91` |
| 警告/取消 | 草莓红 | `#FF6B6B` |

### 5.2 组件风格
- 圆角统一 24rpx
- 阴影：`0 4rpx 16rpx rgba(255, 182, 193, 0.2)`

### 5.4 菜品图片方案
**由于不引入外部图片资源，菜品图片采用**：
- 大号 emoji（字号 80rpx）居中
- 背景为渐变粉色圆形容器
- 简单高效，无需网络请求

## 7. 关键流程

### 7.1 注册流程
1. 用户进入登录页 → 点击「注册新账号」
2. 填写用户名（实时查重）、密码、昵称、选择头像 emoji
3. 提交校验通过 → 写入 `users` → 自动登录 → 跳首页

### 7.2 点单流程
1. 「菜单」Tab 浏览菜品
2. 点击菜品卡片进入详情
3. 点击「加入点单」按钮，份数默认 1
4. 底部弹窗提示「已加入」，并提示「去点单」按钮
5. 切到「点单」Tab → 显示已选菜品
6. 调整份数/备注 → 选用餐时间 → 提交
7. 弹窗显示订单 ID → 「查看订单」跳详情

### 7.3 订单状态流转
- 「待定」订单：可「确认」「取消」
- 「已确认」订单：可「标记上桌」「取消」
- 「已上桌」订单：终态
- 「已取消」订单：终态

## 8. 错误处理

| 场景 | 处理 |
|------|------|
| 用户名已存在 | 注册时输入框下方红字提示 |
| 密码过短 | 注册按钮置灰 + 提示 |
| 菜品为空 | 菜单页显示空状态插画 + 「添加菜品」按钮 |
| 点单未选菜 | 提交按钮置灰 |
| 订单状态非法流转 | 按钮隐藏 |
| 必填项未填 | 提交按钮置灰 + 输入框红框 |
